DAILY JOURNAL — FIREBASE AND CRUD SETUP SUMMARY

1. Keep the Firebase Admin service-account JSON outside this repository.
2. In Windows Command Prompt, set:

   set "GOOGLE_APPLICATION_CREDENTIALS=C:\full\path\service-account-key.json"

3. Run the independent Firestore test:

   npm run test:firestore

4. Start the Express server:

   npm start

5. In a second terminal, test the complete CRUD API:

   npm run test:api

6. Keep the PERSISTENCE_ID printed by test-api.js. Restart the server and run:

   npm run test:persistence -- <document-id>

Active API endpoints:
- GET    /api/entries
- POST   /api/entries
- PATCH  /api/entries/:id
- DELETE /api/entries/:id

The GET and POST routes satisfy the core assignment. PATCH and DELETE provide
complete CRUD for the extra-credit requirement.

See README.md for complete setup, API bodies, testing, screenshots, and
credential-security guidance.
